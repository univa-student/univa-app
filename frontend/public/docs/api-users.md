# Користувачі

Модуль користувачів у Univa покриває акаунт, профіль, персональні налаштування і зв'язки з навчальним середовищем. Frontend очікує стабільний контракт по даних профілю, тому зміни тут потрібно робити обережно.

## Основні сутності

- користувач як обліковий запис
- профіль як публічні та навчальні метадані
- налаштування як персональні параметри інтерфейсу та поведінки

## Типовий UI-сценарій

1. Користувач входить у систему.
2. SPA завантажує профіль.
3. Дані відображаються в дашборді, спільноті та налаштуваннях.
4. Зміни зберігаються через профільні або settings-ендпоінти.

## Рекомендації по розширенню

- нові поля профілю додавай узгоджено в model, resource і frontend types
- не дублюй ті самі мапери в різних модулях
- якщо поле впливає на UI, онови типи і місця споживання одночасно

## Перевірка після змін

- сторінка профілю відображає нові дані
- налаштування коректно зберігаються та перевантажуються
- ресурси не ламають існуючі клієнтські очікування
