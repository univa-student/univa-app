# Webhooks

Webhooks дозволяють будувати зовнішні інтеграції навколо подій Univa. Це корисно для університетських систем, внутрішніх сервісів або командних автоматизацій, які мають реагувати на зміни без ручного втручання.

## Коли webhooks доречні

- створення або оновлення дедлайнів
- зміни у файловому просторі
- події в груповому workspace
- інтеграція з внутрішніми системами факультету

## Базові правила використання

- підписуй кожен endpoint відповідально
- логуй доставку подій і помилки
- обробляй повторні доставки ідемпотентно
- не роби критичну логіку залежною від одного зовнішнього споживача

## Що передбачити в реалізації

- перевірку підпису або секрету
- таймаути і повторні спроби
- моніторинг доставлених подій
- версіонування формату навантаження, якщо контракт еволюціонує

## Рекомендація для команди

Якщо webhook стає джерелом бізнес-критичного потоку, задокументуй події окремо і не змінюй їхню структуру без явної міграції клієнтів.
