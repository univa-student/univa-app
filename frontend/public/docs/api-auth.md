# Автентифікація

API Univa побудований на Laravel Sanctum з cookie-based авторизацією для SPA. Це означає, що frontend працює через захищену сесію, а не через довільне збереження токенів у клієнтському коді.

## Як працює потік входу

1. Клієнт отримує CSRF cookie.
2. Користувач проходить логін.
3. Браузер зберігає сесію і надсилає cookie на захищені запити.
4. SPA працює з API як з авторизованим користувачем.

## Що важливо для frontend

- не змінюй форму auth-відповідей без потреби
- не ломи cookie-flow новими заголовками або нестандартним storage
- тримай авторизаційну логіку в наявних модулях, а не розкидуй по сторінках

## Що важливо для backend

- повертай стабільні структури відповідей
- валідуй CSRF і доступ до приватних ресурсів
- не змішуй auth-логіку з доменною бізнес-логікою

## Типові помилки

- відсутній CSRF cookie перед логіном
- невірні `SANCTUM_STATEFUL_DOMAINS`
- різні домени frontend і backend без коректної конфігурації
- спроба використовувати локальний storage там, де потрібна сесія браузера
